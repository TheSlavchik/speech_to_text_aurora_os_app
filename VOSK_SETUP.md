# Подключение офлайн-распознавания (Vosk) — что нужно доложить в проект

Код распознавания уже написан (C++ класс `SpeechRecognizer` + поток с Vosk,
живая запись с микрофона, потоковое распознавание, сохранение WAV и текста в
заметку). Но две вещи — саму модель и нативную библиотеку Vosk — нельзя
"написать в QML", это бинарные артефакты. Их нужно один раз положить в проект.

## 1. Модель распознавания (обязательно)

1. Скачай `vosk-model-small-ru-0.22`:
   https://alphacephei.com/vosk/models  →  `vosk-model-small-ru-0.22.zip` (~45 МБ)
2. Распакуй.
3. Положи папку целиком, НЕ переименовывая, в каталог `models/` проекта:

   ```
   speechtotext/
     models/
       vosk-model-small-ru-0.22/
         am/  conf/  graph/  ivector/  README   ...
   ```

При сборке она устанавливается в
`/usr/share/ru.omstu.voicenotes/models/vosk-model-small-ru-0.22` — оттуда приложение её
и грузит (путь задан по умолчанию в `SpeechRecognizer`).

## 2. Библиотека libvosk.so под aarch64 (обязательно)

Vosk — это нативная библиотека на базе Kaldi. Нужен файл `libvosk.so`, собранный
под архитектуру устройства/эмулятора Авроры.

Положи его в `vosk/lib/libvosk.so`:

```
speechtotext/
  vosk/
    vosk_api.h        (уже в проекте)
    lib/
      libvosk.so      <-- сюда
```

Где взять `libvosk.so`:

- Готовые сборки Vosk для Linux: https://github.com/alphacep/vosk-api/releases
  - для устройства (ARM64) — `vosk-linux-aarch64-*.zip`
  - для эмулятора (x86_64) — `vosk-linux-x86_64-*.zip`
  Внутри архива есть `libvosk.so` (и `vosk_api.h`).
- ВАЖНО: собирать надо под ту же архитектуру, что и комплект сборки в Qt Creator.
  Если собираешь под `aarch64` — нужен aarch64-вариант; под эмулятор x86_64 —
  соответствующий. Если готовая сборка не слинкуется/не запустится из-за версии
  glibc, библиотеку придётся собрать из исходников внутри сборочного окружения
  Авроры (Aurora Platform SDK / mb2), см. README в репозитории vosk-api.

Библиотека ставится в `/usr/share/ru.omstu.voicenotes/lib`, приложение находит её через
rpath (задан в .pro).

Валидатор Авроры запрещает пакету «предоставлять/требовать» сторонние
библиотеки. Для приватной libvosk.so это уже обойдено в spec:

```
%define __provides_exclude_from ^%{_datadir}/%{name}/lib/.*\.so.*$
%define __requires_exclude ^lib(vosk|atomic)\.so.*$
```

Это убирает ошибки вида «Provides ... is not allowed» и «Dependency is not
allowed» (libvosk.so, libatomic.so.1).

## 2a. libatomic (зависимость libvosk)

libvosk зависит от libatomic.so.1, которой по умолчанию нет на устройстве.
Симптом при запуске:
`error while loading shared libraries: libatomic.so.1: cannot open shared object file`.

Валидатор Авроры запрещает пакету любые внешние зависимости (даже именованную
`Requires: libatomic`). Поэтому libatomic.so.1 нужно приложить ВНУТРЬ пакета,
как и libvosk.so. Настоящую aarch64-версию берём из репозитория Авроры.

Как достать файл (через эмулятор):

1. Доустанови libatomic на эмулятор:
   ```
   sfdk -c "device=Aurora OS эмулятор 5.2.0.180" device exec -- devel-su pkcon -y install libatomic
   ```
2. Узнай реальный путь файла:
   ```
   sfdk -c "device=Aurora OS эмулятор 5.2.0.180" device exec -- bash -lc 'ls -l /usr/lib64/libatomic.so*'
   ```
3. Скопируй сам бинарник (не симлинк) в проект — cat идёт по симлинку и отдаёт
   настоящие байты; redirect делается на маке:
   ```
   sfdk -c "device=Aurora OS эмулятор 5.2.0.180" device exec -- cat /usr/lib64/libatomic.so.1 > vosk/lib/libatomic.so.1
   ```
4. Проверь, что это ELF (первые байты 7f 45 4c 46) и размер в десятки КБ:
   ```
   xxd vosk/lib/libatomic.so.1 | head -1
   ls -l vosk/lib/libatomic.so.1
   ```

После этого `.pro` (guard `exists($$PWD/vosk/lib/libatomic.so.1)`) сам положит её
в пакет и заставит приложение грузить её из своего каталога через rpath, чем
закрывается и зависимость самой libvosk. Никаких Requires в spec не нужно.

## 3. Сборка

Открой проект в Aurora Qt Creator и собери как обычно. В `.pro` уже добавлены:
- `QT += multimedia` (запись с микрофона);
- линковка `-lvosk` и rpath;
- установка модели и библиотеки в пакет.

Версия пакета поднята до 0.3.

## 4. Разрешение на микрофон

В `ru.omstu.voicenotes.desktop` уже указано `Permissions=Audio;Microphone;UserDirs`.
При первом запуске система может спросить доступ к микрофону — разрешить.

## Как это работает

- Модель грузится один раз в фоне при открытии экрана записи (индикатор загрузки).
- Кнопка микрофона стартует реальную запись (16 кГц, моно). Показывается уровень
  сигнала, длительность и живой (промежуточный) текст.
- «Стоп» — поток аудио дораспознаётся, текст финализируется, запись сохраняется в
  WAV в каталоге данных приложения, создаётся заметка (заголовок + дата + текст +
  аудио) и открывается её просмотр.
- Пустая/шумная запись без слов не создаёт заметку — показывается сообщение.
- Поиск по заметкам, экспорт текста (в файл/буфер) и воспроизведение аудио — как и
  раньше.
